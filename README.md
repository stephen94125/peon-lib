# Peon Framework

[🇨🇳 简体中文](README.zh-CN.md) | [🇬🇧 English](README.md)

> **An enterprise-grade, zero-trust AI Agent Workspace built in Rust.**
> 
> Peon provides a robust sandbox where LLM capabilities are strictly gated by true operating system and personnel-level permission checks.

## 🚀 The Vision

While other AI frameworks blindly execute `bash` commands generated by models, Peon operates on a strict **"prove it before you touch it"** principle. The LLMs running in Peon cannot hallucinate access to your system. Every file read, and every script execution is cross-referenced with your defined RBAC/ABAC models before execution.

By decoupling the AI interaction layer from the system execution layer, Peon achieves true **Defence in Depth**.

## 📦 Workspace Structure

This repository is a Cargo Workspace containing the following crates:

- **[`peon-core`](peon-core/README.md)**: The heart of the zero-trust engine. Includes tools, scanning architecture, and Casbin access enforcement. **(Start here to use the framework!)**
- **[`peon-cli`](peon-cli/README.md)**: Universal CLI entry point to bridge your terminal usage and pipelines (`echo "data" | peon-cli`) with the AI core.
- *`peon-discord / peon-telegram` (Planned)*: Chat platform integrations.

## 🧠 Provider Agnostic

Through our underlying foundation, Peon supports a unified interface over almost any modern AI provider without changing your code:

Anthropic · Azure · Cohere · Deepseek · Gemini · Groq · Huggingface · Hyperbolic · Llamafile · Mira · Mistral · Moonshot · Ollama · OpenAI · OpenRouter · Perplexity · Together · xAI

Switch models instantly by simply configuring your `.env`:
```bash
DEFAULT_PROVIDER=gemini DEFAULT_MODEL=gemini-3.0-flash
```
